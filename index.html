Apredendo git e git hub

  • oque é o git : é um sistema de controle de versão de código, permite registrar mudanças no códigos e manter eles organizados e facilmente indentificaveis, ele mantem um histórico completo de todas as auterações que foram feitas no codigo.
  • git hub é: a rede social com os projetos enviado via git.
  • comitar é : gerar uma nova versão do seu projeto.
  • branch é : criar uma ramificação uma nova linha da atual versão do código para fazer modificações sem alterar a versão principal master, após as alterações feitas na branch se (quiser) fazer marge para salvar as modificações.
  • merge : é juntar a ramificação branch a ramificação master

Comandos do git

  • git init: é para inicializar um repositorio local da pasta atual no git
  • git status: verifica o status atual do repositório, incluindo arquivos modificados e arquivos na area de stage
  • git add: Add arquivos selecionados para área de stage (pronto para ser commitado) é necessário antes de fazer o commit no caso de ser o primeiro commit
  • git add . : Add todos os arquivos do projeto no git
  • git commit -m "escrever alguma coisa": Cria uma nova versão do projeto
  • git log Ver commits recentes
  • git log --all Ver todos os commits
  • git checkout : para voltar, usar uma versão anterior commit. usar o id do commit para voltar para ele. exemplo: git checkout 178bda8d369d3fa093b64f398b2003cc82a535d8
  • git checkout master : voltar para a ultima versão master
  • git checkout -b (nome da branch) : Cria uma branch apartir do commit q se está, e navaga pra ela.
  • git branch : Ver quais ramificações existem atualmente.

Observações

Quando usamos o git log, aparece a palvra HEAD e uma seta.

Isso indica qual commit estamos trabalhando e qual ramificação estamos por exemplo:(master) é a principal.
